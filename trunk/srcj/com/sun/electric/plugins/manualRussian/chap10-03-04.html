Внутри описания ячейки, экспорты объявляются строкой "E". Все экспорты сортируются по названию. Синтаксис:

E<name> | <TD> | <originalNode> | <originalPort> | <flags> | [ | <variable> ]*
<name>название экспорта.
<TD>текстовое описание для записи порта (описано ниже).
<originalNode>название экземпляра узла в этой ячейке, к которому принадлежит экспорт.
<originalPort>название порта на экземпляре экспортируемого узла (может быть пустым если нет выбора).
<flags>флаги для экспорта (см. ниже).
<variable>список переменных для экспорта (см. Раздел 10-4-1).

<flags> поле имеет формат:

<characteristics> [ /A ] [ /B ]

<characteristics> тип экспорта. Выбирается из следующих вариантов:

"U" неизвестно - unknown.
"I" вход - input.
"O" выход - output.
"B" двунаправленный - bi-directional.
"P" питание - power.
"G" земля - ground.
"C" синхроимпульс - clock.
"C1" синхроимпульс - clock phase 1.
"C2" синхроимпульс - clock phase 2.
"C3" синхроимпульс - clock phase 3.
"C4" синхроимпульс - clock phase 4.
"C5" синхроимпульс - clock phase 5.
"C6" синхроимпульс - clock phase 6.
"RO" reference output.
"RI" reference input.
"RB" reference base.
/A показывает что экспорт всегда отображается
/B показывает что экспорт body-only (нет эквивалента на иконке)

Пример:

    Es[18]||conn@14|a|D5G2;|I/B

Экспортирует порт "a" экземпляра узла "conn@14"■ и называет его "s[18]". Текст экспорта закреплен в центре порта ("D5") и его размер 2 еденицы ("G2;"). Экспорт типа input, и появляется только в содержимом (не иконка).

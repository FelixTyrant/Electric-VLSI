Переменные могут быть прикреплены к любому объекту в базе данных Electric. Они расположены в конце многих линий в файле. Когда в объекте описано больше одной переменной, они сортируются по названию. Синтаксис:

<name> ( <TD> ) <type> <value>
<name>название переменной.
<TD>текстовое описание (когда переменная видимая).
<type>тип прикрепленных данных.
<value>данные. Если начинается с "[", то это массив вида [ <переменная1> , <переменная2> , ┘ ]

<name> and <value> поля могут быть заключены в кавычки. Символ обратной косой черты может использоваться для обозначение специальных символов.

<type> поле может быть одним из этих: "B" Boolean ("T" или "F")
"C" Ячейка (формы : ).
"D" Double.
"E" Экспорт (формы : : ).
"F" Float.
"G" Long.
"H" Short.
"I" Integer.
"L" Название библиотеки.
"O" Название инструмента.
"P" прототип Примитивного Узла (формы <технология> : ).
"R" прототип Дуги (формы : ).
"S" String.
"T" Название технологии.
"V" Point2D (формы <x> / <y>).
"Y" Byte (0-255).

примеры:

    ART_message(D5G8;)StxArray4x4B

Добавляет переменную "ART_message" со значением string "txArray4x4B". Текстовое описание показывает центрированный текст ("D5") высотой в 8 едениц ("G8;").

    ART_degrees()F[0.0,3.1415927]

Добавляет переменную "ART_degrees" с массивом двух значений типа float (числа с плавающей точкой): 0.0 и 3.1415927.

    EXPORTS()E[ccc:gate;1{sch}:a,"ccc:hate;1{sch}:b[0:4]"]

Добавляет переменную "EXPORTS" с массивом из двух экспортов ячейки "ccc:gate;1{sch}": "a" и "b[0:4]".

    ATTR_z0(D5G0.5;NPY1;)I50

Добавляет аттрибут "z0" с целым значением 50. Он отображается закрепленным в центре ("D5"), высотой 0.5 едениц ("G0.5;"), записанным как "название=значение" ("N"), является параметром ("P"), и он смещен на 1 по Y ("Y1;").

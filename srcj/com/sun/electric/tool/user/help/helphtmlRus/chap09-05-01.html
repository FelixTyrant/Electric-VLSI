Electric поддерживает работу внутреннего симулятора, IRSM, который использует RC модель для симуляции транзисторов на вентильном уровне. Исходный дистрибутив Electric также не включает IRSM симулятор. Jar файл этого симулятора можно скачать с Static Free Software (подробнее о установке добавочных компонентов Section 1-5).

Для проведения симуляции текущей ячейки, используйте команду IRSIM: Simulate Current Cell меню (Tool / Simulation (Built-in)). После выполнения команды, для работы с полученной информацией откроется окно симуляции. (подробнее об окне симуляции Section 4-12-1). Для генерации входного файла IRSM симулятора, без проведения непосредственно симуляции, используйте команду IRSIM: Write Deck.... Для проведения симуляции по входному файлу используйте команду IRSIM: Simulate Deck....

Пока IRSM устройство запущено, вы можете подавать управляющие воздействия и получать незамедлительно результат. (подробнее о работе с IRSM Section 4-12-1). Ответьте, вы можете сохранять на диске сформированные вами входные импульса в IRSM "command file", с помощью команды (Save Stimuli to Disk...меню Tool / Simulation (Built-in)). Этот файл также можно редактировать в ручную.

Свойства "Simulators" (меню File / Preferences..., "Tools" закладка, "Simulators"), предлагают некоторые настройки IRSIM. Основные настройки разобраны в главе Section 4-12-1.

IRSM использует параметризованный файл для описания измерений и паразитной информации. Файлы ("scmos0.3.prm" и "scmos1.0.prm"), идут вместе с дистрибутивом Electric, но вы также можете создать свой собственный файл и указать IRSM использовать его. В добавок, в этом файле можно указать какую именно модель будет использовать IRSM. По дефолту стоит RC модель, линейную модель также можно использовать.

Продвинуты пользователи могут добавлять свои собственные команды в специальные команды IRSM отладки. Данные команды представляют набор флагов для установки режима отладки. Пункты закладки "IRSIM Debugging" и есть набор этих флагов.

Ниже приведены еще две IRSM настройки.

  • "Show IRSIM commands(показывать список команд)" требует от системы листинга всех команд используемых во время симуляции.
  • "Use Delayed X propagation(задержки распростронения)" менее консервативный, но потенциально более четкий метод определения времени, требуемого для рапростронения неопределенного значения (X) по ИС. Данный метод расчета задержки времени распространения сигнала особо оправдан при использовании асинхронных схем.

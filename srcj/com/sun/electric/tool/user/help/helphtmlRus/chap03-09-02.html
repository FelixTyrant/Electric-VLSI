Open Library... команда (в меню File) открывает новую библиотеку в Electric c диска. Эти библиотеки могут иметь расширение ".elib" or ".jelib" (формат jelib - по умолчанию см. Section 10-1).

Вы можете также использовать открытый-библиотечный значок от области инструмента.

Пользователи Electric с очень старыми расширениями ".elib" могут иметь трудность. Сохранение ".elib" от версии 7.00 будет работать должным образом в существующей системе.

By default Electric searches for libraries in the working directory, absolute file path references, and Electric's internal library directory. Пользователи могут добавить, дополнительные директории, чтобы искать, используя "LIBDIRS" находящуюся в рабочей директории. Она определяет дополнительные пути, чтобы искать библиотечные файлы. Файл имеет следующий синтаксис:

     * <comments>
     include <another_LIBDIRS_file>
     <library_directory>

Пути могут быть абсолютны или относительны.

Кроме того библиотеки Electric , читают описания схемы, которые находятся в других форматах с помощью команды File / Import меню:

  • CIF (Caltech Intermediate Format) используется, чтобы описать рисунок схемы, но ничего не известно о транзисторах и контактах: только уровни. Вы можете использовать экстрактор точки , чтобы конвертировать CIF к компонентам Electric. (см.Section 9-10-2). Просмотреть как читается CIF:(в меню File / Preferences..., "I/O" section, "CIF" tab). см. Section 7-3-2 больше о CIF.
  • GDS II (Stream) также используется, чтобы описать рисунок схемы, но ничего не известно о транзисторах и контактах: только уровни. Вы можете использовать экстрактор точки , чтобы конвертировать GDS к компонентам Electric. (see Section 9-10-2). Просмотреть как читается GDS, используйте (в меню File / Preferences..., "I/O" section, "GDS" tab). см. Section 7-3-3 больше GDS.
  • EDIF (Electronic Design Interchange Format) используется, чтобы описать оба схемных решения и уровни. Electric читает EDIF версия 2 0 0. Просмотреть как читается EDIF: (в меню File / Preferences..., "I/O" section, "EDIF" tab) (см. Section 7-3-4).
  • LEF (Library Exchange Format) формат обмена, который описывает элементы в библиотеке. Элементы, которые содержатся в портах, но очень краткое содержание.
  • DEF (Design Exchange Format) является форматом обмена, который описывает содержание библиотеки.Часто использует связанные файлы LEF, которые уже читались. как читается DEF, используйте (in menu File / Preferences..., "I/O" section, "DEF" tab) (см. Section 7-3-5).
  • DXF (AutoCAD) iформат обмена моделирования, и это может содержать трехмерные объекты, которые могут не читаться в Electric. Однако, Electric создает библиотеку примитивов фотошаблона. формат обмена моделирования, и это может содержать трехмерные объекты, которые могут не читаться в Electric. Однако, Electric создает библиотеку примитивов фотошаблона.Как читается DXF, используйте (в меню File / Preferences..., "I/O" section, "DXF" tab) (см. Section 7-3-7).
  • SUE (Schematic User Environment) является схематическим редактором, который фиксирует единственный элемент в каждом файле. Электрическая схема обращается к файлам, добавленным к текущей библиотеке вместо того, чтобы быть помещенным в ее собственной библиотеке. Как читается SUE, используйте (в меню File / Preferences..., "I/O" section, "SUE" tab) (см. Section 7-3-8).
  • ELIB является старшим библиотечным форматом, который находится в неописанном двоичном{бинарном} формате.
  • Readable Dump i старший библиотечный формат Electric, который фиксирует данные в читаемом текстовом формате. Эти файлы использовались, когда картотека ".elib" была основным способом сохранить библиотеки, потому что путь был необходим от файлов библиотеки.Теперь, когда более новый формат ".jelib" также читаем текстом, нет никакой потребности это использовать.
  • Text Cell Contents используется, чтобы читать текстовый файл в текстовых разделах. Текущим окном должно быть текстовое вид (типа VHDL, Verilog, документации, и т.д.) см.Section 4-10.

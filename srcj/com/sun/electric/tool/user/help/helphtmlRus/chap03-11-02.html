При редактировании одного вида элемента, есть команды в View в меню, которая коммутирует к дополнительному виду того же самого элемента.
  • Use Edit Layout View коммутировать к виду размещения.
  • Use Edit Schematic View коммутировать к схематическому виду.
  • Use Edit Icon View коммутировать к виду Значка.
  • Use Edit VHDL View коммутировать к виду VHDL.
  • Use Edit Documentation View коммутировать к текстовому документационному виду.
  • Use Edit Skeleton View коммутировать к Скелетному виду.
Для всех других типов вида, используют Edit Other View... и выбирают желательный вид. Заметьте, что эти команды эквивалентны Edit Cell... команда (в меню Cell) с соответствующим отбором.

При редактировании элементов с текстовыми видами (VHDL, Документацией, и т.д.), окно становится текстовым редактором Вы можете использовать Text Cell Contents... команда (в меню File / Export and File / Import) сохранять и восстанавливать этот текст на диск. См Section 4-10.

Команды, чтобы редактировать другие виды работают только, когда этот элемент существует. Чтобы создавать новый элемент, используйте Make... команда View в меню. Вот эти команды изменения вида:

  • Make Icon View создает значок от схемного решения (см. Section 3-11-4.
  • Make Schematic View создает schematic из layout.
  • Make Alternate Layout View... измененные от layout or schematic к дополнительному layout. Вы должны выбрать удельную технологию layout, и новое layout будет использовать элементы от той технологии.
  • Make Skeleton View делает skeletonized layout из layout.
  • Make VHDL View cконвертирует текущее layout или schematic в структурный VHDL. Этот VHDL используется СБИС-транслятором (см Section 9-12) и ALS симулятором (см Section 9-5-2). Заметьте, что есть 5 схематических примитивов, которые могут существовать в нормальном и инвертированном виде ("buffer", "and", "or", "xor", and "mux"). Вы можете выбрать названия, чтобы использовать для этих двух форм в разделе "Schematics" предпочтения "Technology" (в менюFile / Preferences..., "Technology" раздер, "Technology" tab).

Создание и удаление ячеек Узлов

узлы - компоненты технологии, и они создаются из геометрических фигур на слоях. Для редактирования существующего узла, выберите его из ячейки explorer или с помощью команды Edit Cell... (в меню Edit).

Для того чтобы создать новый узел, используйте пункт контекстного меню "TECHNOLOGY NODES" ячейки explorer и выберите "Add New Node". Узел может быть удален просто с удалением его ячейки. Узел может быть переименован с переименованием его ячейки, но не забудьте использовать слово "node-" перед именем узла (то есть старое имя "node-metal" а новое имя "node-metal-1"). Наконец, вы можете перестроить порядок, в котором узлы будут перечислены с помощью команды "Reorder Nodes" контекстного меню.

Редактирование Специальной информации узла

Ячейка узла сожержит 4 изображения узла внизу и текстовую информацию над ними. Вы можете обновлять пункты текстовой информации дважды щелкнув по ним.

Пункт "Serpentine transistor" указывает, что это - транзистор МОП - структуры, и может требоваться произвольная информация контура, чтобы описать ее геометрию.

Пункт "Square"вынуждает узел иметь одинаковые значения X и Y для измерения.

Пункт "Invisible with 1 or 2 arcs" указывает на то, что узел не будет нарисован если он точно не подключен к одной или двум линия соединения. Это полезно в схематическом изображении выводов, которые видимы только когда не соединены или образуют соединение 3 или больше проводов.

Пункт "Lockable" указывает , что этот узел может быть создан неизменным наряду с другими блокируемыми примитивами, когда блокировка включена в течение редактирования (см Section 6-2 для того чтобы узнать больше о блокировке этих примитивов). Это обычно используется в технологиях массива типа FPGA (см Section 7-6-2).

Пункт "Function" описывает функцию узла, которая отличается от функций линии соединения и слоя. Далоговое окно предлагает список возможных функций узла.

Редактирование Геометрии Узла

Для узлов, принято зарисовывать 4 различных примера узла в различных шкалах, так, чтобы X и Y, масштабирующие правила могут быть получены (квадратные узлы, нуждались только в двух примерах). Если только один пример определен, предполагаются правила масштабирования по умолчанию.

Маленький пример, названный главный пример, используется как заданный по умолчанию размер и также содержит всю специальную информацию порта. Само собой разумеется, важно сохранить геометрию каждого примера далеко от других так, чтобы редактор технологии мог отличить их.

Каждый пример должен содержать те же самые геометрические слои (только зарисованные). Как в ячейках линий соединения, части геометрии могут быть созданы, с помощью выбора из составляющего меню боковой панели, создание геометрии, и затем дважды щелкая, чтобы назначить слой. Если любая многоугольная геометрия используется (например, пункт Filled Polygon, шестой сверху), требуется, чтобы была назначена информация контура (см Section 6-10-1). Если выбран пункт Opened circle arc (второй снизу), вы можете определить количество градусов круга с помощью команды Object Properties... (в меню Edit / Properties).

Каждый пример должен также содержать подсвеченный слой для того, чтобы указывать правильное подсвечивание на экране. Выберите пункт "HIGH" составляющего меню для создания специального типа слоя.

Каждый пример должен содержать информацию порта. Выберите пункт "PORT" в составляющем меню для создания специального типа слоя. Вам нужно будет задать имя для каждого порта, и имя должно быть одинаковым на каждом примере.

Порты на главном примере должны также иметь информацию обеспечения связи (какие линии соединения могут соединиться с ними) и распологать информацией (допустимый угол связанных линий соединения). Дважды щелкните, чтобы настроить.

Диапазон состоит из двух чисел: угол (в градусах против часовой стрелки с 3 часов) и угловой диапазон. Например, угол порта 90 с диапазоном угла порта 45 описывает порт, который указывает вверх и может соединиться под углами до 45 градусов от этого направления. Диапазон будет графически изображен.

Порты на главном примере должны также указать любое внутреннее электрическое обеспечение связи, фактически подключая их вместе. Например, два поликремниевых порта на МОП транзисторе должны быть связаны в главном примере. Присоединитесь к портам с помощью универсальной линии соединения. Не помещайте это внутреннее подключение в любой другой пример кроме главного. Для того чтобы посмотреть местоположения всех портов на главном примере, используйте команду Identify Ports (в меню Edit / Technology Specific).

Так же, как и с линиями соединения, используйте команду Identify Primitive Layers для обозначения каждого куска геометрии в главном примере.

Специальные Рассмотрения Узла

Есть некоторые специальные случаи, доступные в описаниях узла. Часть геометрии в главном примере может быть изменена (дважды щелкая на его функции) к "SET-MINIMUM-SIZE". Это указывает на то, что текущий размер наименьший, какой возможен, и это не может масштабироваться меньше (это используется "mocmos" технологией для слоя металлизации в контактах). Ограничение может быть удалено с помощью описания "CLEAR-MINIMUM-SIZE". Эта опция не может использоваться в змеиных транзисторах.

Другой специальный случай в описании узла - способность определить, что множитель вырезает уровни. Если большие примеры имеют больше вырезанных слоев, правила получены для интервала вырезки и сдвига так, чтобы произвольные числа вырезки могли быть вставлены как ответственные масштабы.

Хотя змеиные транзисторы МОП - особый случай, они не могут быть автоматически идентифицированы, но должны быть явно обозначены с текстовым индикатором. Помимо этой явной индикации, узел транзистора должен содержать четыре порта: два на слое затвора (поликремний) и два на управляемом слое (активном). Должна использоваться стандартная геометрия что показывает поликремний и диффузии, пересекающееся в центральной области транзистора. Любое отклонение от этого формата может заставить редактора технологии быть неспособным получить змеиные правила для узла.

Помимо стандартных узлов для транзисторов, контакты, и другие элементы схемы, необходимо формировать узлы чистого слоя и вывод. Должен быть один вывод для каждой линии соединения, так, чтобы она могла соединиться с другими ее типа. Вывод должен быть создан из псевдослоев (то есть это не имеет никакой реальной геометрии), и должен иметь функцию "вывода", а также один порт в центре, который соединяется с одной линией соединения. Редактор технологии выдаст предупреждение, если нет никакого узла вывода, связанного с линией соединения.

Узлы чистого слоя должны также быть сформированы, один для каждого слоя. Они должны иметь только одну часть геометрии и иметь функцию "pure-layer". Редактор технологии выдаст предупреждение, если нет никакого узла чистого слоя, связанного со слоем

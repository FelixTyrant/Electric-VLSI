Набор электрически связанных компонентов определяет network . Сети могут охватить много дуг, или они могут постоянно находиться на только единственном{отдельном} экспорте на единственном узле. Поскольку сети сохранены в базе данных Electrc, к ним можно немедленно обратиться когда необходимо.

Всякий раз, когда порт на узле выбран, подсвечивание указывает полную сеть, которая связана с тем портом. Другой способ видеть полную сеть состоит в том, чтобы использовать , Show Network команда (в меню Tool / Network). Это подсветит все дуги в настоящее время выбранных сетях. Повторное использование этой команды заставляет сеть быть подсвеченной на последовательно более низких уровнях иерархии.

Если проект очень плотен, Вы можете выбрать одну или более сетей по имени Select Object... команда (в меню Edit / Selection ).

Резистор может быть обработан как соединение или несоединение узла. По умолчанию, это не подключает сети на ее двух концах, так что идентификация степени сети заканчивается в резисторе. Чтобы игнорировать резисторы и обрабатывать их как провода, использование "Network" предпочтение (в меню File / Preferences... , раздел "Tools", вкладка "Network"), и проверка "Ignore Resistors". Тогда подсвеченные сети пройдут через них. Cм раздел Раздел 7-5-1 больше о резисторах.

Другой контроль в "Network" обсужден в другом месте.see Section 6-9-3). Для объяснения входов в "Node Extraction" раздел, см Section 9-10-2).

Есть много команд в меню Tool / Network, которые могут использоваться, чтобы получить информацию о сетях в ячейке:

  • List Networks показывает список сетей в текущей ячейке.
  • List Exports on Network списки весь экспорт называют текущей подсвеченной сетью. Этот список содержит названия{имена} экспорта на всех уровнях иерархии, выше и ниже текущей ячейки. Средство полезно, если, например, Вы размножили линии всюду по схеме и желаете удостовериться, что все экспортные названия{имена} на этой сети имеют названия{имени} "phi". Быстро исследуя этот список, Вы можете увидеть все названия{имена}, которые использовались на сети, всюду по иерархии.
  • List Exports below Network Этот список подобен тому, произведенному List Exports on Network за исключением того, что это работает только на ячейках ниже текущего.
  • List Connections on Network списки - все узлы в текущей ячейке, которые связаны с текущей сетью. Этот список включает только те узлы в конце сети, или узлы контакта, используемые в сети. Команда полезна, если Вы - в одном конце провода и хотите проверить{отметить}, чтобы видеть то, что - в другом конце.
  • List Geometry on Network списки - вся геометрия в текущей ячейке, которая связана с текущей сетью. Они сообщают об области и периметре всех приложенных уровней.
  • List Total Wire Lengths on All Networks перечисляет длины всех сетей в текущей ячейке.
